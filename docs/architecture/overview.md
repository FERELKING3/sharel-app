# Architecture - Vue d'ensemble

SHAREL est structuré autour d'une séparation claire UI / logique métier / services :

- `view/` : widgets et pages (UI) — utilise `go_router` pour la navigation.
- `viewmodel/` : logique d'état (Riverpod `StateNotifier` / providers) — MVVM simplifié.
- `services/` : couches techniques (ex: `ShareEngine`, `permission_service`).
- `screens/` : écrans hiérarchiques exposés par le routeur.
- `model/` : entités immutables (Freezed generated).

Diagramme simplifié (ASCII):

  +-----------------+    +-----------------+    +---------------+
  |     Screens     | -> |   ViewModels    | -> |   Services    |
  +-----------------+    +-----------------+    +---------------+
          UI                     Logique               I/O

Conventions principales

- Pattern : MVVM (Views -> ViewModels -> Services) avec `flutter_riverpod`.
- Routes : centralisées dans `lib/core/router/app_router.dart`.
- Thème & design system : défini dans `lib/core/theme/design_system.dart`.
