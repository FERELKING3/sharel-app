# Transfert - Vue d'ensemble

Le mécanisme de partage utilise un serveur HTTP embarqué (`ShareEngine`) sur l'appareil hôte pour fournir les fichiers sélectionnés. Le client consomme `/session` puis télécharge chaque item via `/file/<index>`.

Modes opérationnels supportés aujourd'hui (implémentation actuelle) :

- `Sender = Host` (par défaut) : l'envoyeur crée la room et sert les fichiers.
- `Receiver = Client` : le récepteur rejoint l'URI de l'envoyeur et télécharge.

Cas recommandés à documenter (non-implémentés / roadmap) :

- `Receiver = Host` : scénario inverse où le récepteur héberge et l'envoyeur pousse les fichiers.
- Multi-client : gestion de plusieurs clients connectés simultanément (à définir).
